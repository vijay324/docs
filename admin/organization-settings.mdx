Organization settings are available to the Admin. The page shows the organization profile, current plan, usage, and links to administrative modules.

## Organization profile

The page displays the organization name, slug, type, industry, size, country, timezone, website, description, and work mode. These identity fields are read-only in the current client interface. Admins can upload or replace the organization logo.

The organization slug appears in signed-in URLs and saved links.

## Use Settings as the administration hub

The Settings page is a read-only organization overview plus links to the administrative surfaces that own each configuration. It shows plan status, billing cycle, renewal information when available, usage, organization identity, and trust-center indicators.

Use the appropriate linked page for changes rather than expecting all settings to be editable in one form:

- **Team management:** invitations, profiles, roles, and Project access.
- **Workflow configuration:** Task status colors, priority colors, story-point scale, and departments.
- **Attendance and leave policies:** organization-level working and time-off rules.
- **Labels:** approved Task label names and colors.
- **Scheduled reports:** recurring analytics email delivery on Max.
- **Billing:** subscription actions and usage review for the Admin.

## Upload an organization logo

Admins can upload or replace the organization logo from the profile area. Use a clear square or horizontal mark that remains recognizable at small sizes. The logo is stored as an organization asset and is used where Flotick displays organization identity.

## Review usage before a plan change

Before inviting members, creating Projects, uploading large files, generating AI content, or downgrading a plan, review the usage cards. These report the organization total, not only the currently signed-in member's activity.
