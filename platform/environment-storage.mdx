---
title: "Environment Storage"
description: "Project-linked environment configuration storage and workspace isolation."
---

`/{orgSlug}/env-storage` provides a structured model for managing environment data by project.

## Data hierarchy

```txt
Category -> Workspace -> Configuration entries
```

## Use cases

- Store environment-specific settings for multiple delivery environments.
- Attach config models to projects for operational traceability.
- Support upload/download operations through dedicated backend endpoints.

## API coverage

- Environment routes are available in the v1 API surface.
- Org-scoped aliases are also exposed under `/api/v1/o/:orgSlug/...`.

## Recommendations

- Restrict edit capability to trusted roles.
- Keep secrets in secure backend-managed storage only.
- Maintain change discipline for production workspace updates.
