## Tenant URL model

Tenant application routes use:

```txt
/{orgSlug}/...
```

Examples:
- `/{orgSlug}/dashboard`
- `/{orgSlug}/tasks`
- `/{orgSlug}/attendance`
- `/{orgSlug}/settings/billing`

## Why this model

- Keeps tenant context explicit in navigation and API calls.
- Reduces accidental cross-tenant data leakage.
- Simplifies debugging, observability, and support operations.

## API alignment

Org-scoped backend aliases exist under:

```txt
/api/v1/o/:orgSlug/*
```

These routes resolve organization membership and role checks before domain handlers execute.

## Client behavior

- Organization state is persisted client-side for continuity.
- Slug mismatch handling redirects users to the correct organization URL.
- Protected layouts enforce auth and onboarding before granting tenant access.
