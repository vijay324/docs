---
title: "Running payroll"
description: "End-to-end checklist for a clean monthly payroll run in Flotick."
---

A monthly payroll run only takes a few minutes when prerequisites are clean. This is the checklist Admins follow.

<Note>
Payroll is Admin-only. Members see their own payslips and loans in the same area.
</Note>

## Before the run (5–10 minutes)

<Steps>
  <Step title="Confirm attendance is closed">
    Members should have checked out for every working day. Use the **Attendance team overview** to spot blanks and ask owners to backfill.
  </Step>

  <Step title="Approve all pending leave">
    Open **Leave → Team requests**. Approve everything pending so attendance reflects approved leaves.
  </Step>

  <Step title="Verify compensation">
    For new joiners or any role changes, confirm **Member Compensation** has the right structure, monthly gross, basic salary, and bank details.
  </Step>

  <Step title="Confirm active loans">
    Open the **Loans** tab. Every loan that should deduct this month must be `Active` with the right `Recovery start` month.
  </Step>
</Steps>

## Create and review the run

<Steps>
  <Step title="Create the run">
    Open **Payroll → Runs** and click **New run**. Pick the **month** and **year**. Flotick rejects duplicates — only one run per period.
  </Step>

  <Step title="Calculation runs">
    Flotick computes per-member earnings, deductions, LOP, OT, statutory amounts, and loan EMIs in parallel. The run lands in `draft` status with one **Payroll Entry** per member.
  </Step>

  <Step title="Review entries">
    Open the run. Sort and filter to spot unusual numbers:

    - Any zero net pay?
    - LOP days that look wrong?
    - Loans deducting that shouldn't be?

    [IMAGE: Payroll run draft with filterable entries table]
  </Step>

  <Step title="Approve">
    When the team agrees, click **Approve**. Status moves through `pending_approval → approved`.
  </Step>

  <Step title="Finalize">
    Click **Finalize**. Status becomes `finalized` (immutable) and **payslips** are generated for every member.
  </Step>
</Steps>

## After finalization

- Members see their payslips in the **Payslips** tab and can download
- Use your bank or accounting integration to disburse — Flotick generates the data, but actual payment runs through your existing channels
- Leave the run in `finalized` for audit; never edit historic runs

## Common gotchas

<AccordionGroup>
  <Accordion title="Wrong LOP days for a member">
    LOP comes from attendance and approved leave. Backfill any missing attendance and re-create the run if you haven't finalized yet.
  </Accordion>

  <Accordion title="Member missing from the run">
    Likely no active **MemberCompensation** assigned. Set compensation, then create a fresh run for that period (you can't add members to a finalized run).
  </Accordion>

  <Accordion title="Loan didn't deduct">
    Check the loan's `Recovery start month`/`year` and `status: Active`. Loans approved but not yet active won't deduct.
  </Accordion>
</AccordionGroup>

## Tips

- Run a **draft** early in the month; review with finance; finalize only after agreement.
- Communicate the finalize date to the team so they can flag corrections.
- Save the date — Flotick lets you finalize once. Anything wrong after that is corrected in the next month's run.
