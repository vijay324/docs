## Onboarding goals

- Give every user a clear role and responsibility.
- Create one source of truth for active work.
- Establish daily routines early to prevent tool drift.

## Recommended rollout order

### Step 1: Admin setup

- Confirm organization profile and settings.
- Configure attendance and leave policies.
- Review billing plan and feature access.

### Step 2: Team structure

- Add departments and invite managers.
- Invite employees by team.
- Validate role permissions before launch.

### Step 3: Delivery setup

- Create initial projects.
- Configure sprint cadence.
- Define workflow statuses and priorities.

### Step 4: Operations setup

- Enable attendance behavior for all users.
- Confirm leave request and approval flow.
- Set payroll structures before first run cycle.

### Step 5: Launch and reinforce

- Use dashboard reviews in daily standups.
- Keep task ownership strict and visible.
- Resolve blockers in real time from board/activity views.
