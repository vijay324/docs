## HTTP conventions

- `2xx`: success
- `4xx`: client/auth/validation/permission issues
- `5xx`: server or dependency failures

## Structured diagnostics

Flotick middleware and logging stack include:
- Correlation/request IDs
- Standardized error handling
- Centralized logging with severity levels

## Common failure classes

- Invalid or expired auth session
- Missing org membership for scoped route
- Permission denied for role-protected action
- Rate limit exceeded
- Validation errors on payload shape and required fields

## Debugging workflow

1. Capture request ID and route.
2. Inspect backend logs and metrics around same timestamp.
3. Confirm dependency health (`/health`, `/api/v1/health`).
4. Reproduce with least-privilege test account to validate authorization logic.
