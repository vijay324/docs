title: "Authentication"

## Member authentication

- Auth endpoints are mounted at `/api/auth/*`.
- Members authenticate with a **JWT access token** (sent as a `Bearer` header) plus an **HttpOnly refresh-token cookie**.
- Refresh tokens use **family rotation** — reusing a rotated token revokes the entire family.
- `requireAuth` middleware resolves the member and their organization context for each request.

## Socket authentication

- Request a short-lived socket token: `GET /api/v1/auth/socket-token`.
- The token authorizes realtime subscriptions over Socket.io.

## Authorization

- **Roles**: `Admin`, `Manager`, `Member`. Admins and Managers have elevated capabilities; exactly one Admin per organization.
- **Org membership** is enforced on every org-scoped route.
- **Per-member permission overrides** (Pro/Max) can fine-tune access within a role's ceiling.

## Production recommendations

- Serve over HTTPS only; refresh cookies are `Secure`, `HttpOnly`, `SameSite`.
- Rotate signing secrets (`JWT_ACCESS_SECRET`, `JWT_REFRESH_SECRET`) on a schedule.
- Keep the frontend and backend `JWT_ACCESS_SECRET` in sync, or refresh validation fails silently.
