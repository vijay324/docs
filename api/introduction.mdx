Flotick exposes a versioned REST API. Most product surfaces in the app are built on the same endpoints.

## Base patterns

- **Versioned REST**: `/api/v1/*`
- **Org-scoped aliases**: `/api/v1/o/:orgSlug/*` — resolve organization membership and role before the handler runs
- **Auth endpoints**: `/api/auth/*`
- **Platform-admin API**: `/api/v1/platform-admin/*`
- **GraphQL** (optional, selected reads): `/api/v1/graphql`

## Core domains

- **Delivery**: `projects`, `tasks`, `sprints`, `dashboard`, `analytics`, `search`
- **Workforce**: `attendance`, `leave`, `payroll`, `holidays`, `shifts`, `members`, `policies`
- **AI**: `ai` (summaries/enhance), `orbit` (insights, chat, agent)
- **Platform**: `organizations`, `polar` (billing), `webhooks`, `feedback`, `contact`

## Conventions

- **Tenancy**: every org-scoped request is bound to one organization; data is never returned across tenants.
- **Files**: uploads and downloads use short-lived presigned URLs rather than proxying through the API.
- **Realtime**: live updates arrive over Socket.io; long-running flows stream over Server-Sent Events.

## Response shape

```json
// Success
{ "data": { } }

// Error
{ "error": { "code": "ERROR_CODE", "message": "Human-readable message" } }
```

See [Authentication](/api/authentication) for tokens and [Error model](/api/error-model) for failure handling.
