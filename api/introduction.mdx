## Base patterns

- Versioned REST base: `/api/v1`
- Better Auth handlers: `/api/auth/*`
- GraphQL endpoint: `/api/v1/graphql`
- Org-scoped aliases: `/api/v1/o/:orgSlug/*`
- Platform-admin APIs: `/api/v1/platform-admin/*`

## Core domains

- Delivery: `projects`, `tasks`, `sprints`, `dashboard`, `analytics`, `search`
- Workforce: `attendance`, `leave`, `payroll`, `employees`, `policies`
- Platform: `organizations`, `subscription`, `webhooks`, `feedback`, `contact`

## Integration surfaces

- File storage endpoints (`/files`) for uploads and entity attachments.
- AI endpoints (`/ai`) for generation and summaries.
- Realtime via Socket.IO + short-lived socket token flow.

## Compatibility notes

- Some domains expose both global and org-scoped entry points.
- Webhooks requiring signature validation use raw request body handling.
