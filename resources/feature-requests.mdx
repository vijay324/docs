---
title: "Feature requests"
description: "Submit product ideas and bug reports, then track their status."
---

Open **Feature Requests** from the app support surfaces. The page shows requests you submitted.

<Steps>
  <Step title="Create a request">
    Click **New request** and add a title, description, and category.
  </Step>
  <Step title="Add evidence">
    Attach up to five PDF, PNG, JPEG, WebP, or GIF files, with a 10 MB maximum per file.
  </Step>
  <Step title="Track the request">
    Status can move through `Open`, `In review`, `Planned`, `In progress`, `Completed`, or `Rejected`. Rejected requests can include a reason.
  </Step>
</Steps>

Categories include new feature, enhancement, integration, user experience, performance, security, and other. You can edit your own request after submission.

For account, billing, or urgent service issues, use [Support](/resources/support) instead of a feature request.

## Write an actionable request

Use a specific title that describes the outcome, then explain the problem, who it affects, and the workflow you expect. For a bug, include steps to reproduce, the page where it happened, the expected result, and the actual result. Attach screenshots or recordings only after removing secrets and personal data.

## Understand request status

| Status | Meaning |
| --- | --- |
| **Open** | Flotick received the request. |
| **In review** | The team is evaluating the problem or idea. |
| **Planned** | The work has been selected for future delivery. |
| **In progress** | Implementation is underway. |
| **Completed** | The request has been addressed. |
| **Rejected** | It will not be pursued in its current form; a reason can be shown. |

You can edit a request you submitted when the action is available. Editing helps correct facts or add clarity; it does not change a request's status by itself.
