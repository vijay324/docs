| Term | Meaning |
| --- | --- |
| Organization | The top-level workspace for members, work, policies, and billing. |
| Member | A person with access to an organization. |
| Project | A container for delivery work, team, files, environments, and reporting. |
| Sprint | A time-boxed delivery cycle inside one Project. |
| Task | A unit of work with ownership, status, priority, and detail. |
| Individual Task | A Task outside a Sprint, using the three-stage personal workflow. |
| Sprint Task | A Task assigned to a Sprint and the organization Sprint workflow. |
| Story point | A relative estimate used for capacity and velocity. |
| Velocity | Story points completed in a Sprint. |
| Scope event | A recorded change to an active Sprint's Tasks or points. |
| Regularization | A request to correct an attendance record. |
| Comp-off | Leave earned for approved work on an eligible non-working day. |
| Orbit Chat | Read-only AI questions and reports grounded in organization data. |
| Orbit Agent | Approved AI execution of supported workspace changes. |
| Plan lock | Non-destructive restricted access applied when usage exceeds a lower plan. |
| Trash | The 30-day recovery area for supported deleted items. |

## Status language

Task and Project states are not interchangeable. A Task status describes the delivery stage of a single unit of work. A Project status describes the overall container. A Sprint status describes the lifecycle of a time-boxed iteration. Use the entity-specific status when communicating with the team so a Task marked `Done` is not mistaken for a completed Project or Sprint.

## Access language

**Role** is the member's broad organization responsibility. **Permission** is a module operation such as view, create, edit, or delete. **Project membership** controls intended access to a Project. **Plan** is the organization's product entitlement. These controls work together.
