Files can belong to a Project, Sprint, or Task. **All Files** provides an organization-wide browser for files you can access.

## File actions

- Upload one or more files with background progress
- Preview supported formats or download the original
- Add or remove standard tags such as `design`, `specification`, `report`, `draft`, or `final-version`
- Assign an unassigned file to a Project, Sprint, or Task
- Reassign a file to another supported item
- Delete and restore files through Trash

Uploads use short-lived secure upload URLs. Download and inline-view access is also issued on demand rather than exposing a permanent storage URL.

## Limits

Storage and per-file size limits depend on plan. The current implemented maximums are 25 MB on Free, 100 MB on Pro, and 500 MB on Max. The app shows remaining organization storage and rejects files above the active limit before upload.

<Note>
Free organizations can upload Project and Sprint files within plan limits. Task attachments require a plan that enables Task file uploads.
</Note>

## Upload files

You can upload directly from a Task's attachments, a Sprint's attachments, a Project's assets tab, or the All Files workspace. Select the target before uploading whenever possible so the file is immediately attached to the correct work item.

The upload queue continues in the background and displays progress, success, or failure for each file. If an upload fails, retry it after checking your network connection, storage allowance, file size, and access to the target entity.

## Organize the library

In **All Files**, search and filter files, inspect their owner and associated entity, and apply standard tags. Tags help teams distinguish reference material, designs, specifications, project-wide assets, drafts, and final versions without changing the original filename.

Use **Assign** for unassigned uploads and **Reassign** when a file should move between supported Projects, Sprints, and Tasks. Reassigning changes the association, not the file bytes or file name.

## Preview, download, and delete

Use **View** to open a supported file in the browser and **Download** to save the original. Flotick grants the access URL only for the request, so do not rely on a copied storage URL remaining available. Delete a file when it is no longer needed; restore it from [Trash](/features/trash) during the recovery window.

<Tip>
Attach a canonical specification to the Project, a planning artifact to the Sprint, and evidence or implementation context to the individual Task.
</Tip>
