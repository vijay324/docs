Files can belong to a Project, Sprint, or Task. You can also upload an unassigned file and connect it later.

## Browse the library

**All Files** is organized into **All Files**, **Projects**, **Sprints**, **Tasks**, and **Unassigned** tabs. Search by file name, select multiple files, and inspect the uploader and current association.

Available actions include:

- Upload with drag and drop or the file picker
- Preview supported images and PDFs
- Download the original
- Copy a temporary view link when available
- Assign or reassign files to a Project, Sprint, or Task
- Add standard tags such as `design`, `specification`, `draft`, or `final-version`
- Delete files and restore them from [Trash](/features/trash)

## Upload limits

| Plan | Organization storage | Maximum file size |
| --- | ---: | ---: |
| Free | 500 MB | 25 MB |
| Pro | 5 GB | 100 MB |
| Max | 100 GB | 500 MB |

Project, Sprint, Task, and unassigned uploads are available on every plan within these limits. The app shows current storage usage and rejects an upload that exceeds the active allowance.

## Keep files organized

Attach long-lived reference material to the Project, planning material to the Sprint, and execution evidence to the Task. Use **Assign** for an unassigned upload and **Reassign** when its work context changes.

Viewing and downloading use short-lived access URLs. A copied storage URL is not a permanent public link.

<Warning>
Do not upload secrets to Files. Store environment configuration in [Env Storage](/features/env-storage).
</Warning>
