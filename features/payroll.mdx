Flotick's payroll module manages the monthly cycle from salary structure setup to payslip generation. It supports Indian compliance — Provident Fund (PF), Employees' State Insurance (ESI), and professional tax — out of the box.

<Note>
Payroll is an Admin-managed feature. Members see their own payslips and loans only.
</Note>

[IMAGE: Payroll page with tabs for Overview, Compensation, Structures, Runs, Payslips, Loans]

## How payroll fits together

```
Salary structure  →  Member compensation  →  Payroll run
                                                    │
                                                    ▼
                                  Payroll entry per member
                                                    │
                                              approve & finalize
                                                    │
                                                    ▼
                                              Payslips
```

## 1. Define a salary structure (Admin)

A **salary structure** is a reusable template of components.

- **Earnings** — Basic, House Rent Allowance (HRA), Special Allowance, allowances
- **Deductions** — PF, ESI, Professional Tax, custom deductions
- **Statutory toggles** — flags whether each component is subject to PF, taxable, etc.

Mark one structure as the **default** for new members.

## 2. Assign compensation (Admin)

For each member:

- Pick a **salary structure**
- Enter **annual CTC**, **monthly gross**, and **basic salary**
- Add bank details, PAN, and UAN — these are stored encrypted at rest

[IMAGE: Member compensation form with bank details and salary structure selector]

## 3. Create a payroll run (Admin)

<Steps>
  <Step title="Open Payroll → Runs">
    Click **New run** and pick the **month** and **year**. Only one run is allowed per period per organization.
  </Step>

  <Step title="Calculation begins">
    Flotick calculates each active member's pay in parallel:

    - Working days, present days, and **loss-of-pay** days from attendance and leave
    - **Overtime hours** at 1.5× hourly rate
    - **Earnings** per the assigned structure
    - **Deductions** for PF, ESI (if eligible), professional tax (Karnataka slab), TDS
    - **Loan EMIs** for active member loans
  </Step>

  <Step title="Review">
    The run lands in **draft** status with one **payroll entry** per member. Filter, sort, and inspect totals.

    [IMAGE: Payroll run draft with per-member entries and totals]
  </Step>

  <Step title="Approve">
    Click **Approve** when entries look right. Status moves through `pending_approval → approved`.
  </Step>

  <Step title="Finalize">
    Click **Finalize**. Status becomes **finalized** (immutable). **Payslips** are generated for every member via idempotent upsert.
  </Step>
</Steps>

## 4. Members view their payslips

In the **Payslips** tab, members see every payslip generated for them. Click a row for the printable layout, including a downloadable PDF.

[IMAGE: Payslip detail with earnings, deductions, and net pay summary]

## Member loans

Members can apply for loans (advances or formal loans) directly in the **Loans** tab.

| Status | Meaning |
| :--- | :--- |
| **Pending** | Submitted, waiting for Admin approval. |
| **Approved** | Admin approved; not yet disbursed. |
| **Disbursed** | Funds released. |
| **Active** | EMI deductions running through payroll. |
| **Closed** | Fully recovered. |
| **Cancelled** | Cancelled before disbursement. |

EMIs deduct automatically once the recovery month/year is reached and the loan is `active`.

## Statutory rules built in

- **PF** capped at the standard ₹15,000 wage ceiling
- **ESI** applies only when monthly gross is ≤ ₹21,000
- **Professional tax** uses the Karnataka slab
- **Overtime** at 1.5× of the per-hour rate

These rules are encoded in the calculation engine; you don't need to wire them up.

## Common workflows

- **Mid-month joiners**: pro-rated automatically based on join date and attendance.
- **Loss of pay**: per-day rate = `monthly gross ÷ total days in month`. LOP amount = `per-day rate × LOP days`.
- **Bonus or one-off**: add as a structure component or an entry-level adjustment in draft.

## Tips

- Run a **dry calculation** in draft mode first; finalize only after the team verifies entries.
- Use **labels** in salary structure components for clarity in the payslip.
- The finalize step is **irreversible**. If something is wrong, reach out to support — corrections happen in the next month's run.
