A Project is the main container for delivery work. It starts in `Planning` and keeps its own members, visibility, Sprints, Tasks, files, and recurring schedules.

## Project lifecycle

`Planning → Active → Review → Completed`

Authorized members advance a Project one stage at a time from **Project settings**. The underlying lifecycle stays fixed, but you can change the display label, color, and order used for each stage.

| Visibility | Who can open the Project |
| --- | --- |
| **Public** | Organization members with Project access |
| **Private** | Assigned Project members, the creator, and privileged organization roles |

## Project workspace

| Tab | What it contains |
| --- | --- |
| Overview | Progress, health summary, recent work, overdue Tasks, and team context |
| Sprints | Project Sprints and their current lifecycle state |
| Team | Project membership |
| Assets | Files attached to the Project |
| Recurring Tasks | Daily, weekly, or monthly Task schedules |

Use **Analytics** from the Project header to open reporting already scoped to that Project.

## Create a Project

<Steps>
  <Step title="Define the Project">
    Click **New project**, then enter a name and optional description.
  </Step>
  <Step title="Set access">
    Choose **Public** or **Private** and add the members who need to work in it.
  </Step>
  <Step title="Add context">
    Upload reference files or use **Draft with AI** to prepare editable Project content.
  </Step>
  <Step title="Plan delivery">
    Create Tasks, add a Sprint, or configure a recurring Task schedule.
  </Step>
</Steps>

Project creation and team management normally require Admin or Manager access. Plan limits apply to active Projects.

## Manage or remove a Project

Use **Project settings** to edit its name, description, visibility, members, displayed status styling, or lifecycle stage.

Deleting a Project moves it to [Trash](/features/trash) for 30 days and also soft-deletes its Sprints and Tasks. Restoring the Project restores that nested work when plan limits and access rules allow it.

<Note>
A Project affected by a lower plan limit remains retained but can become restricted until the organization restores capacity.
</Note>
