A Project is the main container for delivery work. It has a name, description, readable ID, lead, visibility, members, dates, status, and health.

## Status and visibility

Project status supports `Active`, `On Hold`, `Archived`, `Cancelled`, and `Completed`. Health is tracked as `On Track`, `At Risk`, `Off Track`, or `Unknown`.

- **Private:** available to explicitly assigned members and organization roles with broader access.
- **Public:** visible according to organization access rules.

## Project workspace

| Tab | What it contains |
| --- | --- |
| Overview | Project progress, recent work, and Task summary |
| Sprints | Sprints, dates, capacity, status, and velocity |
| Team | Project membership |
| Analytics | Project metrics for a selected time range |
| Assets | Project files and environment workspace access |
| Recurring Tasks | Daily, weekly, or monthly Task schedules |

Admins and Managers can create Projects and manage Project membership. Project settings also allow authorized members to edit details and customize the display label, color, and order of Project statuses.

Archived Projects remain available without counting as active Projects. Deleted Projects move to [Trash](/features/trash).

## Create a Project

<Steps>
  <Step title="Open Projects">
    Click **New project**. Creating a Project normally requires Admin or Manager access.
  </Step>
  <Step title="Define the Project">
    Enter its name and optional description, select visibility, lead, dates, status, and health, then choose the members who need direct access.
  </Step>
  <Step title="Set up delivery">
    Add Tasks, create a Sprint, upload reference assets, and create recurring Task schedules where appropriate.
  </Step>
</Steps>

The Project receives a short ID based on its configured prefix. That ID is stable for links and references even if the Project name later changes.

## Manage the Project team

Project membership is separate from organization membership. Add members who need Project access and remove members when their work moves elsewhere. Admins and Managers have broader organization access, but a private Project still uses explicit membership to control the intended working group.

## Archive, complete, and delete

Use **Completed** when delivery is finished and **Archived** when the Project should no longer be active but needs to remain visible. Both states preserve the Project record. Deleting sends the Project to Trash; restore it before the 30-day recovery period ends.

If a Project is plan-locked after a downgrade, it remains visible but becomes read-only until capacity is restored.
