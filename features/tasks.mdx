## Task fields

| Field | Options or purpose |
| --- | --- |
| Type | `Story`, `Bug`, `Task`, `Epic`, or `Spike` |
| Priority | `Critical`, `High`, `Medium`, or `Low` |
| Responsibility | Assignee, reviewer, tester, and reporter |
| Planning | Project, optional Sprint, story points, estimates, due date, and labels |
| Detail | Description, acceptance criteria, considerations, attachments, subtasks, comments, links, and work logs |

## Individual and Sprint Tasks

An **individual Task** is not attached to a Sprint and moves through `To Do`, `In Progress`, and `Completed`.

A **Sprint Task** uses the organization's Sprint workflow. The default stages are `Backlog`, `Sprint Backlog`, `Todo`, `In Progress`, `Review`, `Testing`, `Blocked`, and `Done`.

## Work with Tasks

The Tasks page supports **List**, **Board**, **Calendar**, and single-Project **Backlog** views. You can switch between all, assigned, created, and team scopes when your role permits.

Search and filter by status, priority, type, Project, Sprint, responsibility, label, and date. Select multiple visible Tasks to change status, priority, or Sprint, or to delete them in bulk.

Open a Task to manage its content and history. Status and assignment changes are recorded. Add work logs in hours to update time spent.

<Note>
Tasks in completed Sprints are excluded by default. Enable **Show completed sprint tasks** when you need historical work in the current view.
</Note>

Deleting a Task moves it to [Trash](/features/trash). Access to create, edit, assign, or delete depends on role, Project access, and any member-specific permission override.

## Create a Task

<Steps>
  <Step title="Choose where the Task belongs">
    Click **Create task** from Inbox, Tasks, a Project, or a Sprint. Select a Project first. Add a Sprint only when the Task is planned for that iteration.
  </Step>
  <Step title="Set the work type and urgency">
    Enter a title, then choose type and priority. Use `Epic` for a larger unit of work and link the related Tasks rather than relying on naming alone.
  </Step>
  <Step title="Set responsibility and planning data">
    Assign an owner, reviewer, tester, and reporter as needed. Add story points, estimates, labels, and a due date when those help the team plan or report work.
  </Step>
  <Step title="Save and add detail">
    Save the Task, then use the detail page to add acceptance criteria, considerations, files, subtasks, links, comments, and work logs.
  </Step>
</Steps>

## Change status safely

On a board, drag a card to a permitted column. In list and detail views, use the status control. Flotick validates the target state against the active workflow. If a move is rejected, check the transition rules, required stage order, or any configured work-in-progress limit with an Admin.

Individual Tasks and Sprint Tasks intentionally behave differently. Adding a Sprint changes the Task into Sprint work and makes the Sprint workflow available. Removing it returns the Task to Project backlog handling where permitted.

## Bulk actions

Select visible rows in the list to update multiple Tasks at once. You can set a common status, priority, or Sprint, or soft-delete the selected Tasks. Flotick reports any rows that could not be changed so you can resolve access or workflow conflicts individually.

## Time, history, and dates

Use **Log work** to record hours and an optional description. Time spent accumulates on the Task and contributes to eligible reporting. Status history captures the previous and new status, actor, and time; assignment history records changes to each responsibility. Completion uses a stable completion timestamp so later edits do not change the reporting period in which the work was finished.

Due dates power Calendar view, overdue filters, deadline analytics, and Orbit risk questions. They do not automatically prevent a Task from moving to a later status.
