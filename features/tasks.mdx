Tasks are where the actual work lives. Every project's progress reduces to the state of its tasks.

[IMAGE: Tasks page with List, Board, and Calendar view tabs]

## What a task contains

| Field | Description |
| :--- | :--- |
| **Title** and **description** | Title is plain text; description supports Markdown and is encrypted at rest. |
| **Type** | `Story`, `Bug`, `Task`, `Epic`, or `Spike`. |
| **Priority** | `Critical`, `High`, `Medium`, `Low`. |
| **Status** | The workflow stage (see below). |
| **Assignee, Reviewer, Tester, Reporter** | Up to four people with different responsibilities. |
| **Story points** | Estimation in points. |
| **Estimates and time spent** | Original estimate, remaining estimate, and accumulated time spent. |
| **Labels** | Free-form text tags. |
| **Acceptance criteria, considerations** | Long-form fields, encrypted. |
| **Due date** | Used for deadline alerts and the calendar view. |
| **Sprint** | Optional sprint membership. |
| **Files**, **Comments**, **Work logs**, **Links** | Built-in collaboration around the task. |

## Workflow status

Tasks flow through a default workflow, which Admins can adjust per organization (see [Workflow policy](/admin/workflow-policy)).

```
Backlog → Sprint Backlog → Todo → In Progress → Review → Testing → Blocked → Done
```

Drag-and-drop on the board moves a task between columns. Each transition is validated server-side:

- Required transitions (e.g., must pass through `Review` before `Done`) are enforced
- WIP limits per column are respected — if a column is at capacity, the move is rejected
- Reaching `Done` triggers a confetti animation in the UI

## Create a task

<Steps>
  <Step title="Open the new task dialog">
    Click **New task** from any list, board, or project page. The dialog opens immediately.
  </Step>

  <Step title="Fill the essentials">
    Title, project, type, and priority are required. Optionally set assignee, reviewer, tester, story points, labels, and due date.
  </Step>

  <Step title="Save">
    The task starts in `Backlog`. The assignee sees it on their [Personal Board](/features/personal-board) instantly via real-time updates.
  </Step>
</Steps>

## Update a task

Open any card to bring up the **task detail dialog**. From here you can:

- Change status, priority, assignee, and dates
- Add or update **acceptance criteria** and **considerations**
- Attach files (drag-drop)
- Post threaded **comments** and `@mention` teammates
- Log **work** (hours and description)
- Add **dependencies** (see [Task dependencies](/features/task-dependencies))

[IMAGE: Task detail dialog with tabs for Details, Comments, Files, Worklog, Links]

## Time tracking

Two ways to log time:

1. **Manual work log** — open the task, choose **Log work**, enter hours and description
2. **Focus session** — start a [Focus Session](/features/focus-sessions) tied to the task; on stop, Flotick auto-creates a work log

Either way, `Time spent` rolls up automatically.

## History and audit

Every task carries a complete history:

- **Status history** — when the status changed and who changed it
- **Assignment history** — when the assignee, reviewer, or tester changed

Open the **History** tab on the task to review.

## Encryption

Long-form, sensitive fields are encrypted with AES-256-GCM at rest:

- Description
- Acceptance criteria
- Considerations
- Optionally, comment content

Encrypted fields are decrypted only on read for users with access. They aren't included in full-text search.

## Soft delete

Deleting a task moves it to [Trash](/features/trash) for 30 days. Comments, work logs, history, and links are preserved during the recovery window. After 30 days, hard deletion runs automatically.

## Tips

- Use **labels** for cross-cutting tags (`platform`, `cust:acme`, `tech-debt`) since they're free-form and indexable.
- Save effort estimates — sprint velocity and AI summaries get sharper with consistent estimation.
- Promote a task to an **Epic** when scope expands; link sub-tasks to it (see [Task dependencies](/features/task-dependencies)).
