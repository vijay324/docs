## Comments

Tasks and Sprints support comments. On Tasks, you can:

- Write Markdown
- Reply in a thread
- Like or unlike a comment
- Edit your comment
- Soft-delete a comment you are allowed to remove

Comments display their author, time, edited state, replies, and likes. Real-time updates keep open workspaces current for connected members.

## Subtasks

Open a Task's **Subtasks** section to create a checklist item with a title and optional description. You can complete, reopen, reorder, edit, or delete subtasks according to your Task permissions.

Subtask completion is tracked separately from the parent Task. Completing every subtask does not automatically move the parent Task to `Done`.

## Write a useful comment

Open the Task or Sprint, scroll to **Comments**, and write the update. Markdown is supported for headings, emphasis, lists, links, code, tables, and task lists. Submit the comment to make it visible to other members with access to the work item.

To continue a focused conversation, choose **Reply** on a comment. Replies remain grouped beneath the parent comment. Use likes to acknowledge an update without adding another message.

## Edit and remove comments

Your edited comment retains an edited indicator. Soft-deleting a comment removes it from the normal conversation while preserving the record for authorized recovery and audit behavior. Editing or deleting another member's comment depends on the access rules applied to that Task or Sprint.

## Manage subtasks

<Steps>
  <Step title="Add the checklist item">
    In **Subtasks**, enter a concise title and optional description.
  </Step>
  <Step title="Order the work">
    Reorder items to represent the expected execution sequence.
  </Step>
  <Step title="Track completion">
    Check an item when it is complete. Reopen it when more work is needed.
  </Step>
</Steps>

Subtasks work best for a small, concrete checklist. When an item needs its own owner, due date, review, or discussion, create a linked Task instead.
