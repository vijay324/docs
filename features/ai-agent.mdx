---
title: "Orbit Agent"
description: "Turn a plain-language instruction into a reviewed sequence of workspace changes."
---

Orbit Agent is visible to Admins and Managers from the **Agent** tab in Orbit. Each action is still checked against the requesting member's role and access.

## Run an instruction

<Steps>
  <Step title="Describe the outcome">
    Name the relevant Project, Sprint, Tasks, members, dates, and status changes.
  </Step>
  <Step title="Review the plan">
    Orbit resolves the referenced records and presents every proposed step. Nothing runs before approval.
  </Step>
  <Step title="Approve execution">
    Run the plan and follow each step through pending, running, succeeded, failed, or skipped state.
  </Step>
  <Step title="Review history">
    Open the completed session to inspect the instruction, actions, results, and usage.
  </Step>
</Steps>

## Supported actions

- Create Tasks, Projects, and Sprints
- Update Tasks and Projects
- Assign one Task or assign up to 25 Tasks in bulk
- Change Task status or deadline
- Add a Task comment
- Complete a Project
- Cancel a Task

A plan can contain up to 20 steps. Some actions require Admin access even when the Agent tab is visible; for example, creating or completing a Project and bulk assignment are Admin-only. Other delivery actions require Manager-level authority.

<Warning>
Orbit Agent does not delete records, manage members or billing, change organization settings, or send messages outside Flotick.
</Warning>

## Undo

Admins can undo supported update actions from a completed session when the target has not changed in a way that makes reversal unsafe. Creations and comments are not undoable. Agent warns about newer changes instead of silently overwriting them.

Review every plan for the correct records, responsibility, dates, and status before approval. A failed step can prevent dependent steps from running.
