---
title: "Orbit Agent"
description: "Turn a plain-language instruction into a reviewed sequence of workspace changes."
---

Orbit Agent is available to Admins and Managers from the **Agent** tab in Orbit.

## How execution works

<Steps>
  <Step title="Describe the outcome">
    Ask Orbit to create or update delivery work, assign Tasks, set deadlines, add a comment, complete a Project, or cancel a Task.
  </Step>
  <Step title="Review the plan">
    Orbit resolves referenced Projects, Sprints, Tasks, and members, then presents each proposed step. Nothing runs before approval.
  </Step>
  <Step title="Run">
    Approve the plan. Progress updates in real time and each action records its status.
  </Step>
  <Step title="Review or undo">
    Open a completed session from history. Supported actions can be undone when the original state is still valid.
  </Step>
</Steps>

Agent tools include creating Tasks, Projects, and Sprints; updating Tasks and Projects; assigning one or multiple Tasks; changing status; setting deadlines; and adding comments.

<Warning>
Orbit Agent never deletes records. Removal requests are mapped to safe, visible state changes such as completing a Project or blocking a Task with an audit comment.
</Warning>

A plan can contain up to 20 steps, and bulk assignment is limited to 25 Tasks per action.

## What Agent can do

| Action | Typical use |
| --- | --- |
| Create Project, Sprint, or Task | Turn a delivery plan into structured workspace records. |
| Update Project or Task | Change supported details such as title, priority, status, or planning data. |
| Assign Tasks | Assign one Task or distribute a selected batch. |
| Change Task status or deadline | Move work or set a due date after reviewing the plan. |
| Add a comment | Record a decision or context on a Task. |
| Complete a Project or cancel a Task | Use safe state changes instead of deletion. |

## Review the plan carefully

The plan shows the actions, targets, and resolved members or work items before execution. Confirm the Project, Sprint dates, assignees, task titles, and status changes are correct. Cancel the session if anything is ambiguous; you can submit a clearer instruction in a new session.

After approval, each step progresses through pending, running, succeeded, failed, skipped, or undone states. A failed step can stop dependent remaining steps, so review the session history rather than assuming a partially completed plan performed every change.

## Undo and history

When an action is eligible, Agent exposes an undo control from the completed session. Undo is not guaranteed: it requires the original entity state to still make the reversal safe. Agent history is available in Orbit, letting authorized users inspect instruction, planned work, execution state, and credit use later.

<Tip>
Start with a small instruction—one Project or a few Tasks—before automating a larger workflow. It is easier to verify the resulting pattern and refine the wording.
</Tip>
