---
title: "Task links"
description: "Connect related Tasks and express blocking or duplicate relationships."
---

Open a Task and use **Dependencies** to link another active Task in the same organization.

| Link | Meaning |
| --- | --- |
| `Blocks` | This Task blocks the linked Task. |
| `Blocked by` | This Task is blocked by the linked Task. |
| `Relates to` | The Tasks are associated without a blocking relationship. |
| `Duplicates` | The Tasks represent duplicate work. |

You cannot link a Task to itself or add the same link twice. Each linked Task shows its readable ID, title, type, and current status. Authorized editors can remove links from either Task.

<Note>
Task links communicate relationships; they do not automatically change either Task's status.
</Note>

## Add a link

<Steps>
  <Step title="Open the source Task">
    Go to its detail page and locate the **Dependencies** section.
  </Step>
  <Step title="Choose the relationship">
    Select `Blocks`, `Blocked by`, `Relates to`, or `Duplicates`.
  </Step>
  <Step title="Find the other Task">
    Search for the target Task and confirm the link. Flotick displays the linked Task's ID, title, type, and status after saving.
  </Step>
</Steps>

`Relates to` and `Duplicates` are symmetric when viewed from either Task. A blocking relationship appears as its inverse from the target Task, so a Task that is blocked by another clearly identifies the dependency.

## Manage linked work

Links are useful during planning, triage, and review. Use **Blocks** when work order matters, **Relates to** for context, and **Duplicates** when two records describe the same work. Remove a link from either connected Task when the relationship is no longer useful. Deleting the link does not delete, merge, or change either Task.
