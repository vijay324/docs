title: "Real-time updates"
description: "Keep Tasks, comments, notifications, and Orbit activity current across connected sessions."
---

Flotick refreshes supported organization data when another member changes it. Common live surfaces include Task lists and boards, comments, notifications, Project and Sprint activity, and Orbit Agent progress.

## Connection behavior

- The app authenticates its real-time connection with your current session.
- Short network interruptions trigger automatic reconnection.
- Relevant cached views refresh when a domain event arrives.
- The notification center displays the current connection state.

If a view appears stale, confirm that the bell menu shows **Live**, then refresh the page. Unsaved form input is not shared with other members.

<Note>
Real-time delivery improves freshness; the server remains the source of truth and validates every saved change.
</Note>

## What you will notice

When a teammate updates a Task, moves a board card, posts a comment, creates a Project or Sprint item, or triggers an Orbit Agent action, the affected list and open detail views can refresh without a manual page reload. This reduces the chance of working from stale board state during collaboration.

## If updates stop arriving

1. Open the notification bell and check its connection indicator.
2. Confirm the browser has network access and is not in offline mode.
3. Click the offline indicator to retry the connection.
4. Refresh the affected page if it remains stale.
5. If the problem continues, contact [support](/resources/support) with the organization slug, page, and approximate time.

Real-time updates do not overwrite unsaved changes in an open form. Save or cancel your draft before reloading a page that has changed elsewhere.
