---
title: "Env Storage"
description: "Store encrypted environment configuration by Project, codebase, and workspace."
---

Env Storage is available on Pro and Max. Access also depends on member permission and Project access.

## Structure

`Project → Codebase → Workspace → Environment configuration`

Codebase types include backend, frontend, mobile, database, DevOps, API, and custom. A workspace can represent production, development, staging, testing, or a custom environment, with `Active`, `Inactive`, or `Maintenance` status.

## Manage a workspace

<Steps>
  <Step title="Choose a Project">
    Open **Env Storage** and select an accessible Project.
  </Step>
  <Step title="Create the structure">
    Add a codebase, then create the workspaces it needs.
  </Step>
  <Step title="Add configuration">
    Use **Normal** view to edit key-value rows, or **Developer** view to edit raw `.env` content.
  </Step>
  <Step title="Use the configuration">
    Reveal or copy a value, copy a `KEY=value` pair, copy all, or download the workspace as an `.env` file.
  </Step>
</Steps>

You can import a `.env` or `.txt` file up to 1 MB. Review the target workspace before importing or exporting.

## Versions and encryption

Each saved configuration creates a version you can inspect later. Current values are masked until revealed.

Workspace values are encrypted at rest. You can also enable zero-knowledge encryption with a custom passphrase; the entire configuration is then encrypted and decrypted in your browser.

<Warning>
A zero-knowledge passphrase cannot be recovered by Flotick. Losing it makes the protected configuration unreadable.
</Warning>

Deleting a codebase or workspace removes its nested configuration after confirmation. Keep production and non-production values in separate workspaces.
