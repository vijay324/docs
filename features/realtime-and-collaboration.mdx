## Realtime model

- Socket.IO is used for event distribution.
- Canonical envelope event: `realtime:event`.
- Legacy event names are still emitted for compatibility.

## Delivery guarantees

- Events are emitted to org room `org:{organizationId}`.
- If present, project-targeted fan-out is emitted to `project:{projectId}`.
- Idempotency keys are used to reduce duplicate processing windows.

## Product impact

- Dashboard cards refresh without manual reloads.
- Board transitions propagate instantly to collaborators.
- Notifications and activity states remain current across active sessions.

## Implementation notes

- Client subscriptions are initialized through the shared socket provider.
- Authenticated token/cookie flows are supported for socket auth.
- Realtime behavior can be combined with optimistic UI for responsive interactions.
