The Analytics page at `/<org-slug>/analytics` aggregates the work happening in your organization into clear, scoped dashboards.

<Note>
Available to Admins and Managers. Members see their own [Personal Dashboard](/features/personal-dashboard) instead.
</Note>


## Scopes

| Scope | Where to find it |
| :--- | :--- |
| **Organization** | Analytics page → Organization tab. |
| **Project** | Project detail → Analytics tab. |
| **Sprint** | Sprint detail → Analytics tab. |
| **Member** | Member detail → Analytics tab. |

## Organization dashboard

Headline metrics across the org for a date range you pick (default last 30 days):

- **Task throughput** — created vs completed
- **Completion rate**
- **Active members** — distinct people contributing
- **Attendance summary** — present, late, on leave
- **Leave volume** — by type
- **Trend lines** for each of the above


## Project deep dive

Drill into a single project to see:

- **Sprint contribution** — velocity per sprint
- **Member intelligence** — who's contributing what
- **Task aging** — how long tasks sit in each status
- **Project health** — `On Track`, `At Risk`, `Off Track` history

## Sprint analytics

For each sprint:

- **Burndown chart** vs ideal
- **Velocity** on completion
- **Scope events** timeline
- **Top contributors** in the sprint

## Member analytics

Self-service or admin view of an individual's contribution:

- Tasks completed and points delivered
- Time spent per project
- Attendance and leave summary
- Comment and review activity

Available to the member themselves and to their Admin or Manager.

## How fresh is it

Analytics use a write-behind cache for performance. Cache windows are short (under two minutes for most scopes), and the underlying event pipeline updates aggregates as your team works. Open the page, and what you see is the latest agreed snapshot.

## Plan gating

Some advanced views (e.g., long-range trend exports, detailed bottleneck analysis) are available on Pro and Max plans. See [Plans](/admin/billing/plans).

## Tips

- Filter by **department** to compare teams within the same org.
- Use the **export** action to take a snapshot for an external review.
- Pair analytics with [AI Summaries](/features/ai-summaries) for narrative context on the numbers.
