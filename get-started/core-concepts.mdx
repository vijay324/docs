## Work hierarchy

| Concept | Purpose |
| --- | --- |
| **Organization** | Contains members, policies, work, files, and the subscription. |
| **Project** | Groups a delivery area, its team, Sprints, Tasks, files, and environments. |
| **Sprint** | A time-boxed cycle in one Project. |
| **Task** | A deliverable with classification, status, dates, and collaboration. |
| **Epic** | A Task type that groups direct child Tasks in the same Project. |

Projects, Sprints, and Tasks receive readable IDs for navigation and reference.

## Task responsibility

Sprint Tasks can identify an assignee, reviewer, tester, and reporter. They can also include supported collaborator roles such as approver, watcher, or contributor.

Individual Tasks are lightweight Project work without Sprint responsibility fields. They use `To Do`, `In Progress`, and `Completed`; Sprint Tasks use the organization Sprint workflow.

## Roles

| Role | Typical access |
| --- | --- |
| **Admin** | Organization, billing, policy, member, and work administration |
| **Manager** | Team and delivery management within the role ceiling |
| **Member** | Accessible work and personal attendance and leave |

On Pro and Max, per-member overrides can change eligible operations within a role ceiling. See [Member permissions](/admin/member-permissions).

## How access is evaluated

You must be an active organization member. Flotick then checks plan entitlement, role and overrides, record visibility, and Project membership. Access to the Tasks page does not grant access to every private Project or Task.

Signed-in routes include the organization slug, such as `app.flotick.com/<org-slug>/tasks`.

## Use each record intentionally

Use a Project for a delivery area, a Sprint for a bounded iteration, a Task for a deliverable, an Epic for a one-level work group, a subtask for a checklist item, a comment for discussion, and a file for an artifact.

Status describes workflow position, priority communicates urgency, story points estimate relative Sprint effort, and work logs record time already spent. Keeping these fields current improves Sprints, Analytics, and Orbit.
